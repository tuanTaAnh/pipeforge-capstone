Table meaning:
- dim_customers contains customer identity and segment.
- dim_plans contains subscription plan labels.
- fact_subscriptions contains subscription-to-customer/plan links and MRR amount.
- stripe_invoices contains monthly invoice amounts and discount amounts.
- stripe_payments contains successful collected payment amounts and refund amounts.

Join rules:
- fact_subscriptions.customer_id = dim_customers.customer_id
- fact_subscriptions.plan_id = dim_plans.plan_id
- stripe_invoices.subscription_id = fact_subscriptions.subscription_id
- stripe_payments.invoice_id = stripe_invoices.invoice_id

Metric mapping:
- MRR or gross MRR = sum(fact_subscriptions.mrr_amount)
- billed revenue = sum(stripe_invoices.invoice_amount)
- discount amount = sum(coalesce(stripe_invoices.discount_amount, 0))
- collected revenue = sum(stripe_payments.successful_amount)
- successful payment amount = stripe_payments.successful_amount
- refund amount = sum(coalesce(stripe_payments.refund_amount, 0))
- net MRR = gross MRR - discount amount - refund amount

Dimension mapping:
- customer = dim_customers.customer_id and dim_customers.customer_name
- customer name = dim_customers.customer_name
- customer segment = dim_customers.customer_segment
- plan = dim_plans.plan_id and dim_plans.plan_name
- plan name = dim_plans.plan_name
- invoice month = stripe_invoices.invoice_month
- payment month = stripe_payments.payment_month

Date/month rules:
- invoice_month and payment_month are text values in YYYY-MM format.
- For "May 2026", filter invoice_month = '2026-05' for invoice/billed/MRR-by-invoice-month queries.
- For "May 2026" payment/collection queries, filter payment_month = '2026-05'.
- Do not use DATE() unless the metadata says the column is a date. These month columns are month strings.

Direct analytics patterns:

1. Minimum and maximum successful payment amounts
User intent examples:
- "What are the minimum and maximum successful payment amounts?"
- "Show min and max successful payments."
SQL pattern:
SELECT
  MIN(successful_amount) AS min_successful_amount,
  MAX(successful_amount) AS max_successful_amount
FROM stripe_payments;

Rules:
- Use stripe_payments.successful_amount.
- Do not use payment_amount.
- Do not use status.
- Do not join invoices unless a filter/dimension requires invoice context.

2. Customer with highest MRR
User intent examples:
- "Who is the customer with highest MRR?"
- "Which customer has the most MRR?"
- "Top customer by MRR."
SQL pattern:
SELECT
  c.customer_id,
  c.customer_name,
  c.customer_segment,
  SUM(s.mrr_amount) AS total_mrr
FROM fact_subscriptions AS s
JOIN dim_customers AS c
  ON c.customer_id = s.customer_id
GROUP BY
  c.customer_id,
  c.customer_name,
  c.customer_segment
ORDER BY total_mrr DESC
LIMIT 1;

Rules:
- Do not ask currency clarification.
- Do not filter subscription status.
- Use all subscription rows for MRR because the simplified demo has no status column.

3. MRR by customer segment
User intent examples:
- "Show MRR by customer segment."
- "Break down MRR by segment."
SQL pattern:
SELECT
  c.customer_segment,
  SUM(s.mrr_amount) AS total_mrr
FROM fact_subscriptions AS s
JOIN dim_customers AS c
  ON c.customer_id = s.customer_id
GROUP BY
  c.customer_segment
ORDER BY total_mrr DESC;

4. Billed revenue in a month
User intent examples:
- "What was billed revenue in May 2026?"
- "Show invoice revenue for 2026-05."
SQL pattern:
SELECT
  SUM(invoice_amount) AS billed_revenue
FROM stripe_invoices
WHERE invoice_month = '2026-05';

Rules:
- Use stripe_invoices.invoice_amount.
- Use invoice_month for month filter.
- Do not use issued_at.

5. Collected revenue by payment month
User intent examples:
- "Show collected revenue by payment month."
- "How much successful payment amount did we collect each month?"
SQL pattern:
SELECT
  payment_month,
  SUM(successful_amount) AS collected_revenue
FROM stripe_payments
GROUP BY payment_month
ORDER BY payment_month;

6. Refund amount by payment month
User intent examples:
- "Show refund amount by payment month."
SQL pattern:
SELECT
  payment_month,
  SUM(COALESCE(refund_amount, 0)) AS refund_amount
FROM stripe_payments
GROUP BY payment_month
ORDER BY payment_month;

7. Discount amount by invoice month
User intent examples:
- "Show discounts by invoice month."
SQL pattern:
SELECT
  invoice_month,
  SUM(COALESCE(discount_amount, 0)) AS discount_amount
FROM stripe_invoices
GROUP BY invoice_month
ORDER BY invoice_month;

8. Net MRR by customer segment
User intent examples:
- "Show net MRR by customer segment."
SQL pattern:
WITH invoice_adjustments AS (
  SELECT
    i.subscription_id,
    SUM(COALESCE(i.discount_amount, 0)) AS discount_amount,
    SUM(COALESCE(p.refund_amount, 0)) AS refund_amount
  FROM stripe_invoices AS i
  LEFT JOIN stripe_payments AS p
    ON p.invoice_id = i.invoice_id
  GROUP BY i.subscription_id
)
SELECT
  c.customer_segment,
  SUM(s.mrr_amount) AS gross_mrr,
  SUM(COALESCE(a.discount_amount, 0)) AS discount_amount,
  SUM(COALESCE(a.refund_amount, 0)) AS refund_amount,
  SUM(s.mrr_amount) - SUM(COALESCE(a.discount_amount, 0)) - SUM(COALESCE(a.refund_amount, 0)) AS net_mrr
FROM fact_subscriptions AS s
JOIN dim_customers AS c
  ON c.customer_id = s.customer_id
LEFT JOIN invoice_adjustments AS a
  ON a.subscription_id = s.subscription_id
GROUP BY c.customer_segment
ORDER BY net_mrr DESC;
